UrbanSound8K Khuyến nghị
8,732 audio clips · 10 urban sound classes · 4 giây/clip · SR 22kHz. Cùng task audio classification, nhiều mẫu hơn ESC-50 nhưng ít class hơn — dễ benchmark chéo (train ESC-50, test UrbanSound8K zero-shot).

AudioCaps Audio-Text pairs
~46K clips từ AudioSet + text captions do người viết. Cực phù hợp cho ImageBind vì đây là dataset audio-text alignment — giống đúng task của InfoNCE loss. Có thể dùng để eval Recall@K (text→audio retrieval).

FSD50K Mở rộng
51,197 clips · 200 sound event classes (Freesound). Nhiều class hơn, khó hơn ESC-50, test độ tổng quát tốt hơn. Annotations được kiểm tra kỹ bởi Freesound. Phù hợp nếu muốn benchmark full evaluate.

4.1 Cross-dataset Evaluation Nên làm
Train trên ESC-50 (folds 1-4) → test zero-shot trên UrbanSound8K. Đây là bằng chứng tốt nhất về khả năng generalize của model. Cần viết thêm UrbanSound8KDataset tương tự ESC50Dataset.
GeneralizationZero-shot transfer
4.2 Audio-Text Retrieval (Recall@K) Metric chuẩn
Thay vì chỉ tính accuracy (classification), tính Recall@1, R@5, R@10 cho task retrieval: cho 1 audio query → rank tất cả text, xem text đúng có nằm trong top-K không. Đây là metric chính của ImageBind paper.
RetrievalR@1/5/10Paper metric
4.3 Embedding Visualization (t-SNE / UMAP) Visual đẹp
Lấy audio embeddings của toàn bộ test set → giảm chiều bằng UMAP hoặc TSNE → vẽ scatter plot tô màu theo class. Kiểm tra xem các class có tách biệt rõ trong embedding space không. Code ~15 dòng với umap-learn.
